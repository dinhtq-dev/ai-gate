Hướng dẫn sử dụng

Giới thiệu

AIGate là dịch vụ proxy API phá vỡ giới hạn client, chuyển đổi các model lớn miễn phí như Gemini, Antigravity, Qwen Code và Kiro thành giao diện chuẩn tương thích OpenAI mà mọi ứng dụng có thể gọi.

Truy cập thống nhất

Truy cập nhiều model lớn với một cấu hình qua giao thức chuẩn tương thích OpenAI

Vượt giới hạn

Hiệu quả vượt qua giới hạn tốc độ và hạn mức API miễn phí bằng ủy quyền OAuth

Chuyển đổi giao thức

Hỗ trợ chuyển đổi thông minh giữa giao thức OpenAI, Claude và Gemini

Pool tài khoản

Hỗ trợ luân phiên đa tài khoản, chuyển dự phòng tự động và hạ cấp cấu hình

Sơ đồ quy trình vận hành

1

Cấu hình

Đặt tham số cơ bản trên trang "Cấu hình"

  • Đặt API Key
  • Chọn nhà cung cấp model khởi tạo khi khởi động
  • Cấu hình tùy chọn nâng cao
2

Tạo xác thực

Tạo ủy quyền OAuth trên trang "Nhóm nhà cung cấp"

Cách 1: Ủy quyền OAuth
  • Nhấn nút "Tạo xác thực"
  • Hoàn tất đăng nhập OAuth trong pop-up
  • Xác thực được lưu tự động
Cách 2: Tải lên thủ công
  • Thêm node nhà cung cấp mới
  • Tải lên file ủy quyền hiện có
  • Liên kết thủ công đường dẫn xác thực
Cách 3: Tích hợp API nhà cung cấp
  • Thêm node cho giao thức tương ứng trong "Nhóm nhà cung cấp"
  • Nhập API Key và Endpoint (Base URL)
  • Không cần tạo/tải file xác thực OAuth
3

Quản lý xác thực

Xem và quản lý xác thực trong "File xác thực" (Bỏ qua với nhà cung cấp không OAuth)

  • Xem file xác thực đã tạo
  • Tự động liên kết với provider pool
  • Xóa xác thực không hợp lệ
4

Bắt đầu sử dụng

Xem ví dụ định tuyến trong "Truy cập nhanh" và bắt đầu gọi API

  • Xem ví dụ gọi định tuyến
  • Sao chép địa chỉ endpoint API
  • Cấu hình trong ứng dụng client

Hướng dẫn cấu hình client

Dưới đây là cách cấu hình cho các client AI phổ biến. Đặt endpoint API thành địa chỉ dịch vụ này:

Cherry Studio

  1. Mở Cài đặt → Model Providers
  2. Thêm nhà cung cấp tùy chỉnh
  3. Đặt API URL thành: http://localhost:3000/{provider}
  4. Nhập API Key (REQUIRED_API_KEY từ cấu hình)

Cline / Continue

  1. Mở Cài đặt VS Code
  2. Tìm cấu hình Cline hoặc Continue
  3. Đặt API Base URL thành: http://localhost:3000/{provider}/v1
  4. Nhập API Key và tên model

Gọi cURL chung

curl http://localhost:3000/{provider}/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "ten-model",
    "messages": [{"role": "user", "content": "Hello!"}],
    "max_tokens": 1000
  }'
Mẹo: Thay {provider} bằng đường dẫn nhà cung cấp thực tế, như gemini-cli-oauth, claude-kiro-oauth, v.v. Xem ví dụ định tuyến trên trang "Truy cập nhanh" để biết đường dẫn đầy đủ.

Câu hỏi thường gặp

H: Làm gì khi request trả về lỗi 404?
Đ: Kiểm tra đường dẫn API có đúng không. Một số client tự nối thêm đường dẫn vào Base URL, gây trùng lặp. Kiểm tra URL request thực tế trong console và loại bỏ phần đường dẫn thừa.
H: Làm gì khi request trả về lỗi 429?
Đ: 429 nghĩa là tốc độ request quá cao. Cấu hình nhiều tài khoản trong provider pool với luân phiên; hoặc cấu hình chuỗi Fallback để hạ cấp đa loại.
H: Làm gì khi ủy quyền OAuth thất bại?
Đ: Đảm bảo cổng callback OAuth có thể truy cập (Gemini: 8085, Antigravity: 8086, Kiro: 19876-19880). Người dùng Docker cần ánh xạ đúng các cổng này.
H: Làm gì khi phản hồi streaming bị gián đoạn?
Đ: Kiểm tra ổn định mạng, tăng timeout request của client. Nếu dùng proxy, đảm bảo proxy hỗ trợ kết nối dài.
H: Làm gì khi request trả về lỗi "No available and healthy providers"?
Đ: Nghĩa là tất cả nhà cung cấp loại tương ứng không khả dụng. Kiểm tra trạng thái sức khỏe nhà cung cấp trên trang "Nhóm nhà cung cấp", xác nhận xác thực OAuth chưa hết hạn, hoặc cấu hình chuỗi Fallback để tự chuyển sang nhà cung cấp dự phòng.
H: Làm gì khi request trả về lỗi 403 Forbidden?
Đ: 403 nghĩa là truy cập bị từ chối. Trước tiên kiểm tra trạng thái node trên trang "Nhóm nhà cung cấp". Nếu kiểm tra sức khỏe node bình thường, bạn có thể bỏ qua lỗi này. Nguyên nhân khác có thể gồm: quyền tài khoản không đủ, quyền API Key bị giới hạn, hạn chế truy cập theo vùng, xác thực hết hạn, v.v.
H: Tại sao nên bật "Tự động làm mới OAuth Token"?
Đ: OAuth token thường có thời hạn (vd.: 1 giờ). Khi bật, hệ thống tự làm mới token nền, đảm bảo dịch vụ ổn định 24/7 và tránh request thất bại do token hết hạn.
H: Không bật "Nhà cung cấp model tải sẵn" ảnh hưởng gì đến bảo trì token?
Đ: Hệ thống chỉ tự làm mới cho nhà cung cấp đã khởi tạo trước. Nếu nhà cung cấp chưa được khởi tạo trước, token sẽ không được tự làm mới nền, có thể dẫn đến hết hạn nếu lâu không dùng. Khuyến nghị thêm nhà cung cấp thường dùng vào danh sách khởi tạo trước.
